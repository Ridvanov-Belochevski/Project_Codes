proj_codes documentation¶

Author: Ridwan Bello¶

Consider this hypothetical (but not unrealistic) scenario.

You are an analyst in charge of the portfolio identification for an IEG Evaluation, say, on Transport Decarbonization. Your task is to extract the list of all WB lending projects (excluding Addifitonal Financing), closed and active, approved between FY2010 and FY2020, and mapped to at least one transport sector code in the World Bank Sector and Theme Taxonomy.

You can accomplish this task using one of two platforms: Data Explorer or Data Studio (there may be other ways, but these are the two known to me).

If you wanted to use Data Explorer, you would:

  1. Download the Project Sectors dataset.
  2. Download the All projects dataset.
  3. Merge the two data tables offline.
  4. Then, filter the merged table based on Approval FY, Project status, product type, and sector code.

However, this approach has the following limitations:

  1. It is inefficient: you have to download data you do not need to get to the data that you need.
  2. It is computationally demanding. The Project Sectors and All projects datasets have hundreds of thousands of rows and more that one hundred columns! Working with such large volumes of data is guaranteed to slow down your data software.

On the other hand, if you choose to use Data Studio, you could:

  1. Open up Data Studio via PowerBI
  2. Navigate to the ssrds_Operations dataset.
  3. Create report from scratch.
  4. Create a table visual and drag required variables unto the canvass.
  5. Drag filter variables unto the filter panes, and begin to tick or untick relevant categories.
  6. Export resulting table to Excel.

However, this approach has the following limitation:

  1. It is manual and potentially tedious, if you have to filter data extract by multiple variables and categories.

The proj_codes module addresses these limitations by providing an API that allows users to extract data in 3 steps or lines of code - 4, if users would like to save the data extract. See below.

In [ ]:
#Step 1: import the Sectors object from the proj_codes module, and create a "Sectors" object
from proj_codes import Sectors
demo = Sectors()

Sectors object created.
In [ ]:
#Step 2: load data into the "Sectors" object created
demo.load_data()

#Step 3: extract the required data
#The get_projects method allows users select projects that meet the specified argument criteria 
projs_to_extract = demo.get_projects(sector_codes=['TI','tw','Tv','TP','TC','TF','TZ'],   #Return projects mapped to these sectors
                                      min_pct=20,                  #Only return projects where any code in sector_codes account for 20% or more
                                      start_FY=2010,               #Only return projects approved after FY2010
                                      stop_FY=2020,                #Only return projects approved before FY2020
                                     product_type=['L'],           #Only return lending projects
                                     project_status=['Active','Closed'],   #Only return closed and active projects
                                     include_AF=False,             #Do not include Additional Financing projects in the output
                                     show_meta=True)               #Show additional projects meta-data

497 unique projects meet the specified criteria.

The get_projects method will return all the projects that meet the specified argument. It will also print the number of unique projects that are in the returned data extract (497, in this case).

The user can then save the data extract with the save_last method as shown below.

In [ ]:
#Step 4: Save the data extract
demo.save_last("sector_codes")

Data extract saved.

After calling the save_last method, the data extracted will be saved in the same directory in which the proj_codes is located.

This is the basic functionality of the proj_codes module: allowing users extract and save portfolio data in no more than 4 lines of code.

However, this module has other functionalities that users can use to quickly explore the data they have extracted. For example, users can plot the data extract by the key variables such as sectors, region, approval FY, lending instrument (obviously works for lending projects only), completion status, and lead GP.

See below:

In [ ]:
#Plot the data extract by region.
demo.plot_last("region")
In [ ]:
demo.plot_last("sectors")      #Note that sector mapping are not mutually exclusive
In [ ]:
#The plot_last method accepts a "save_name" argument that lets users save the plot using the value specified as the save name.
demo.plot_last("fy", save_name = "FY_plot")

Plot saved.

Okay. We have used the proj_codes module to extract the projects that are mapped to specified sector codes and that meet other required criteria.

Let us now consider the reverse scenario. Suppose you have a list of projects you have already identified, but you are now just interested in identifying the sectors that these projects are mapped to. Again, to do this, you would have to download the full data from the usual sources, then begin to filter for the projects of interest (or do a VLOOKUP in Excel).

The proj_codes module let's you do that as well. See below:

In [ ]:
#Suppose the user wants to know the sector codes to which these 3 PIDs are mapped
test_pids = ['P055120', 'P072644', 'P079414']
In [ ]:
#The get_sectors function accepts a list of pids and return the sectors that the corresponding projects are mapped to
projs_to_extract_2 = demo.get_sectors(test_pids, show_meta=True)

Data found for 3 out of 3 requested PIDs.
In [ ]:
#The save_last function also accept an optional argument to specify the name the extract should be saved with. See below.
demo.save_last("sectors for PIDs")

Data extract saved.

The proj_codes module provide similar functionality for working with WB Theme codes. The syntax for working with Theme codes is similar to that of sector codes, except for replacing sectors with themes. See below:

In [ ]:
#Import "Themes" (instead of "Sectors") from proj_codes, and create a "Themes" object
from proj_codes import Themes
demo2 = Themes()

Themes object created.
In [ ]:
#Load data into the Themes object created. Note that the syntax is the same with the Sectors class
demo2.load_data()

Loading WB project Themes data.
This typically takes 2-4 minutes. Please wait...
Data loading successful!
Loaded data contains 694863 rows and 121850 unique WB projects.
Total loading time: 71.4 seconds.
Data source: World Bank Standard Reports.
Data download date: May 26, 2022.
In [ ]:
#Suppose you are interested in the theme codes for these 3 PIDs
test_pids = ['P055120', 'P072644', 'P079414']

#The get_themes method accepts a list of PIDs and returns the theme codes that the corresponding projects are mapped to
themes_extract = demo2.get_themes(test_pids, theme_level=[3], show_meta=True)

Data found for 3 out of 3 requested projects.
In [ ]:
#On the contrary, the get_projects accepts a list of theme codes and returns the projects mapped to those themes.
#Suppose user is interested in extracting project mapped to theme codes 811 and 812
test_themes = [811, 812]

#The get_themes functions accepts a list of PIDs and returns the themes that those PIDs are mapped to
project_extract = demo2.get_projects(theme_codes = [811, 812],
                                    min_pct=80,
                                    start_FY=2010,
                                    stop_FY=2020,
                                    product_type=['L'],
                                    project_status=['Closed'],
                                    include_AF=False,
                                    show_meta=True,
                                    show_all=True)

172 unique projects meet the specified criteria..

As with the Sectors class, users can call .save_last() to save the extracted data to their local drives and the .plot_last() to plot the extracted data. Users can also check the documentation to view the full functionality of the Themes class.
